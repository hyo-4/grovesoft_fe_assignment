## 3. 컴포넌트 상세: `Roulette`

이벤트 응모 후 제공되는 룰렛 UI로, 확장성과 안정성을 고려하여 설계했습니다.

```
//Props
rewards: Reward[] : 룰렛에 표시될 보상 목록
disabled: boolean : 응모 전/종료 등 참여 불가 상태 제어
alreadyDone: boolean : (확장용) 이미 룰렛 완료 상태 표시 목적
```

- **SVG 동적 렌더링:** `rewards.length`에 따라 각도를 자동 계산()하여 보상 개수가 변해도 유연하게 대응합니다.
- **정밀한 당첨 로직:** 회전 종료 후 `totalRotation % 360` 연산을 통해 상단 화살표가 가리키는 정확한 보상 인덱스를 산출합니다.
- **UX 안정성:** `isSpinning` 상태값을 활용해 애니메이션 중 중복 클릭을 원천 차단했습니다.

## 4. 기술적 고민과 해결

### ✅ 상태 관리 및 안정성

- **비동기 상태 분리:** `loading`, `error`, `data` 상태를 엄격히 분리하여 데이터 로드 전후의 레이아웃 깨짐을 방지하고, 에러 시 재시도 UI를 제공합니다.
- **메모리 누수 방지:** 카운트다운 구현 시 `setInterval`을 사용하고, 컴포넌트 언마운트 시 `clearInterval`을 통해 리소스 누수를 관리했습니다.

### ✅ 비즈니스 로직 및 유효성 검사

- **단계별 참여 제한:** `LocalStorage`를 활용해 [응모 완료 → 룰렛 활성화 → 참여 완료]의 흐름을 제어하고, 이미 참여한 사용자의 중복 요청을 프런트엔드에서 차단했습니다.
- **실시간 폼 검증:** 정규식을 활용한 연락처/이메일 검증과 필드별 에러 메시지 즉시 노출로 사용자 이탈을 방지했습니다.
